Information Cards - open source http://informationcard.net/taxonomy/term/12/0 Open source software and Information Cards en CardGears Announced - A Simple New Way to Issue Managed Information Cards http://informationcard.net/blog/cardgears-announced <p><img class="image-left" src="/sites/default/files/cardgears-logo.png" alt="" width="250" height="61" />On the ICF mailing list earlier this week, ICF member Markus Sabadello, a leader of several Information Card-related open source projects, announced the availability of <a href="http://www.cardgears.com/">CardGears</a>, a hosted service for web sites wishing to issue Managed <a href="../../../../../../../../quick-overview">Information Cards</a>.</p> <p>As Mr. Sabadello points out, Managed Information Cards can be issued by any website, whether just to provide a simpler and more secure sign-in mechanism, or to expand their brand to becoming part of the user experience every time a card is used. This website issuing a managed card is authoritative for the data on it. Technically, this requires two components:</p> <ul> <li>A card issuing component. This produces and sends to the user a card file (in the .crd format) each time a new card is issued.</li> <li>A Security Token Service (STS). This is the component that provides the claim values (identity information) on a card, such as first name, last name, e-mail address, etc. The STS is invoked every time a user uses or previews their card.</li> </ul> <p>Mr. Sabadello explains, “CardGears makes it as simple as possible to operate both of the above components. First, you can design, issue and modify cards by using the intuitive web interface, without any programming at all. Second, you can use various APIs to integrate the CardGears service with your own applications.
