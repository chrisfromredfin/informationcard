Called CloudCard, it is a standard <a href="../../../../../../../../quick-overview">Information Card</a> selector implementation that requires no installation and works from any conventional browser on a desktop, laptop, or mobile device.</p> <p><img class="image-left" src="/sites/default/files/cloud-selector-icon.png" alt="Cloud selector icon" width="114" height="79" />Susan Morrow, Product Manager for CloudCard, demonstrated today how it eliminates the need for local client software, which is one of the barriers to widespread adoption of the Information Card digital identity standard. CloudCard uses the standard <a href="../../../../../../../../blog/imi-1.0-oasis-standard">IMI 1.0</a> Information Card format and protocol so it works immediately with any Information Card issuer. Websites that wish to accept Information Cards from CloudCard currently need to add some simple custom HTML code to their web page, but according to Ms. Morrow this step is easy compared to the hurdle of requiring users to install a desktop selector, and Avoco plans to standardize this special code so it can be used with any cloud selector.</p> <p>Avoco will demonstrate CloudCard again tomorrow at the Internet Identity Workshop in Mountain View.</p> <p>For more information, see the Avoco Secure <a href="http://www.prweb.com/releases/2100/01/prweb3140224.htm">press release</a> or visit the <a href="https://www.secure2cardspace.com/demo/cloudcard.htm">CloudCard demonstration page</a> on their website.</p> card storage Information Card selector standards Mon, 02 Nov 2009 19:36:36 +0000 Drummond Reed 103 at http://informationcard.net Information Cards at Internet Identity Workshop Spring 2009 http://informationcard.net/blog/news-from-iiw-spring-2009 <p><img class="image-left" src="/sites/default/files/iiw-2009a.png" alt="" width="116" height="116" />The Spring 2009 <a href="http://www.internetidentityworkshop.com/">Internet Identity Workshop </a>concluded three days of intensive discussion of key Internet identity technologies last week.
